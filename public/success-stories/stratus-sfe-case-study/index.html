Case study: Exploring the highly effective partnership of Stratus and the SFDC platform 

Case study: Exploring the highly effective partnership of Stratus and the SFDC platform

by Stratus Platform

Published: Nov 2, 2020

Looking a little more closely at the ways in which Stratus integrates with and provides support for teams; in particular, how Stratus helped Sales Force Enablement team and Salesforce.Com platform build the GCOSC project

“There’s a proven, consistent Pfizer way of building these projects that’s available to all of us and the hard work has already been done. Why not use it?”

About Sales Force Enablement

Sales Force Enablement (SFE) is a functional area in Pfizer Digital which provides support for the field-based medical affairs teams by creating custom tools, applications & processes to manage their data and CRM systems.

In addition, they support a range of other divisions & functional areas across Pfizer (HR, R&D, global product development). Teams like the Core Commercial and Medical Affairs (CCMA) Creation Center team use SFE. They are the most familiar with the platform and function as a Pfizer Centre for Excellence on salesforce.com, functioning both as a delivery team and a platform architecture team.

Challenges and objectives

Historically, workflow tools and development platforms consisted of web or desktop apps in different technologies written in different languages with different standards, because they were built at different times by different contractor teams.

Optimization is key for creating the best solution in a highly-compliant business like Pfizer and, as the SFE platform grew, they started to find use cases that, because of makeup, functionality, data volume and processing, or licensing, needed a more custom solution.

How the solution helped

Partnering with Stratus gave SFE the answer - the custom solutions the SFE team really needed for the consistent needs they were facing around unusual internal-facing web apps, such as the types of services, plus what power/size/quantity, as well as usually VueJS on the client side and PHP/Laravel framework as the server side language, and the standards around Pfizer DevOps process.

A Stratus app can be the UI layer on top of a salesforce API, when using the no/low code Sales Force Dot Com (SFDC) platform doesn’t work for your project. This is a much more user-friendly, straightforward and efficient process than using Heroku or AWS out of the box, and comparatively extremely cost efficient.

Benefits breakdown:

  • Architecture consistency: standardization in how the application is architected and built, the services it uses, the languages it uses, reusable components
  • Process consistency: any rotational contractor coming in is going through exactly the same DevOps process/Travis checks and this standardizes the work. This leads to:
  • Speed: shorter implementation time, easier to estimate, quicker for support to get onboarded
  • Flexibility: ease of the platform and project comms to be agile as it’s all in-house
  • Cost: the licensing and the costs are significantly less than alternative options in the majority of cases
  • Focus on excellence: not building every project from scratch means the team can spend more time on improving the UX of the parts that really matter to the end user
    ​​​​​​​

Example: GCOSC (Global Service Catalogue)

GCOSC is a great example of heavy salesforce usage combined with complex frontend manipulation on Stratus by the development & project team.

GCOSC allows a user to search and select a particular service that they would like to have implemented from a body of catalogues, moving through an official notification and approval process

The service request might be for training, an event, or an additional service, and it follows a different flow for each, and the agent on the salesforce side pulls together the relevant pieces

There are many aspects of personalization in this project that Stratus supports. It does this by providing a stable, basic stack structure, on which the project team can then build the APIs and the rest of the required functionality:

  • Depending on what country the user is in, it will load in different, relevant catalogues
  • Provides multiple pieces of functionality vital to user experience: being able to open a request “on behalf of” another user and - like Jira - being able to “watch” certain service requests
  • Being able to delegate requests to other users so the process isn’t interrupted if the requesting user goes on vacation
  • Being able to favorite requests to save time, as well as add comments, clone a request, add documents, filter by request status, and trace a request’s journey through the system

Stratus is also supporting the Salesforce.com team (SFDC) by providing a structure for the development team to build further UI improvements, which are usually behind feature flag

Finally, Stratus also helps provide Knowledge Articles for users on GCOSC. These are handled through salesforce and are displayed for the user throughout the process; additionally, the agent can even find one from the backend and add it in as a comment to help support the user with managing their service request autonomously. This improves the UX and helps set the user up for success.

Building the new Knowledge Articles process has involved a lot of great communications back and forth between the internal client providing the articles, Stratus finding the best way to upload and handle them in the system, and the SFDC team pulling them through to the front end. It’s a really helpful feature and both SDFC and SFE have learnt a lot - this has been one of the first times Knowledge Articles have been used in this way.

Results/impact/metrics/outcomes

After a successful limited-scope launch in June, the project will be further scaled up starting on November 9th and they are well on their way to meeting KPIs:

  • Number of clients using the client portal: 80 (forecasted after Nov 9th: 300+)
  • Number of requests submitted through the portal: 132 (forecasted after Nov 9th: 2000/month)
  • Number of services offered through the client portal: 8 (forecasted after Nov 9th: 100+)

Future plans
