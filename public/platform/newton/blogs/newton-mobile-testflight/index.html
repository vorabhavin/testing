    Ensuring that a mobile app works in the production environment can be tough!

    Apps have various integrations and connections to different back-end systems throughout the development cycle. This presents various challenges and risks in testing to ensure that any bugs or issues have been caught before a release.

    But there are additional issues with testing our increasingly integrated apps before release. Some app capabilities like “Sign In With Apple” are simply untestable when they are shared from our internal BetaApps testing site. This must be configured in Apple’s production environment to be testable.

    As people who care about releasing a quality app that works, how can we be sure this will work together in the final production environment?

    Enter the Newton Mobile Platform TestFlight integration

    TestFlight is the solution provided by Apple to test an iOS app before release on the public App Store.

    Complexities and limitations in Apple’s code signing and provisioning capabilities have historically prohibited Pfizer teams from taking advantage of this valuable service from Apple while developing mobile apps. It was possible, but required significant manual work fraught with challenges to utilize. It would not scale. The C&CCC Platform DevOps Newton Mobile team recognized this need and set to work finding a way to facilitate the testing of these features before app launch.

    The result? With our new Newton Mobile CI/CD, we’ve automated the processes for code signing and uploading builds of apps to Apple TestFlight! Distributing a pre-release build of your app to stakeholders on Apple’s TestFlight can now be done in a few simple steps as a self-service process!

    Let’s walk through the process…

    Instructions

    All new Newton Mobile Platform projects and apps migrated to Newton Mobile CI/CD support TestFlight. (If you would like to use TestFlight but your project has not yet been migrated to Newton Mobile CI/CD, please contact the Newton Mobile Platform Team for assistance.)

    1. Enter TestFlight metadata in your Newton Mobile Project’s platform_config.json configuration file (click here for example code)

    2. Update testflight_users.csv with name and email address of the testers

    3. The app project team performs a unique Git commit message –testflight –release release to trigger the Mobile Platform TestFlight integration

    4. Users receive an email with an app testing invitation and the Newton Mobile Platform interfaces with the Apple Developer console

    Stakeholder Experience

    Invited testers receive an email with a link to test the app, which directs them to download the TestFlight app before being prompted to install the app to be tested.

    The tested app will appear on the phone’s Home screen (marked with an orange bullet) and in the TestFlight app. The user can test that version of the app for 90 days.

    A Few Tips

    • Your stakeholders will need to install the Apple TestFlight app when accepting their invitation.
    • When they accept an invitation, the Apple ID that is signed in on the iPhone or iPad when they accepted the invitation will be the Apple App Store account that has access to the app (this may be their Personal Apple ID.)
    • Only “@pfizer.com” email addresses or verified vendor email addresses of appropriate stakeholders should be used for pre-release app invite links.
    • Apple reviews all apps that are made available via TestFlight and on the Apple App Store. This review typically takes one day and is only performed on the first submission of a new app version for TestFlight. - Additional Builds of the same app version (e.g. v2.0) don’t need to be reviewed by Apple for subsequent TestFlight distributions. Allow time in your project timeline for this review as well as resolving any issues that may arise.

    Conclusion

    TestFlight is arguably the most reliable manual testing platform to ensure the iOS production environment works end-to-end. Consider taking advantage of this Mobile Platform feature and squash those bugs before release.
