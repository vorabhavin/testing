Auto-Upgrades for Edison Custom sites: keeping a global portfolio of applications up to date

Auto-Upgrades for Edison Custom sites: keeping a global portfolio of applications up to date

by Platform DevOps

Published: Apr 12, 2021

 How Edison is improving support and upgrades for complex website solutions in Pfizer Digital.

The relationship between Edison Custom and Drupal

As the name suggests, Edison Custom is the made-to-measure option for unique website projects. It provides users with the ability to create more complex interactive websites – exactly the type that can’t quite be achieved using Edison’s flexible, out-of-the-box solutions powered by Edison Lite.

Edison Custom is powered by Drupal: a powerful, open-source web content management framework written in a popular language called PHP. Edison Base, the technical name for the base code which makes up Edison Custom, is built on Drupal 9 (the latest version) plus a collection of other software that has been custom-built for Pfizer.

Why this work is essential

Minor releases of Drupal come out every 6 months, and major versions every few years. As with many types of software, when there are new releases, older versions are no longer assured complete security coverage. It is therefore imperative that we stay on current Drupal versions to ensure sites are secure.

Drupal is also heavily augmented (and its functionality extended) by contributed modules from the community and Pfizer’s development teams. These contributed modules are generally not “backwards compatible” – they tend to only work fully on the latest version and are not considered reliable or interoperable with older versions. This can affect a number of different areas, like new features or bug fixes, as well as security. This is a priority for Edison and Pfizer.

The challenges

This is a complex problem to solve as there are a lot of factors:

  • Edison Custom sites are generally complex and often have “snowflake” functionality - each site is very different.
  • Upgrading Drupal and contributed modules can sometimes break sites’ functionality.
  • Historically, projects have had a variable level of automated testing – though Edison continually supported teams over 2020 to increase their test coverage.
  • Project teams have competing priorities. Investing time and money to add further automated testing can get de-prioritized – though of course, ironically, it actually saves time and money in the long run.

Currently, the Operations team runs one large initiative every 6 months to upgrade all the sites to latest Edison Base version. This is a recurring, non-trivial piece of work.

Streamlining the process: Edison Automated Upgrades

​Automated Upgrades is a solution that provides the smart answer, doing exactly what it implies: auto-upgrading Edison Base when required. This means taking advantage of automation to support complex sites, without the historical overheads.

The auto-upgrades process on Edison means automatically updating all Custom sites to the newest version of Edison Base, as soon as it is released, all the way from dev to staging to production environments. It works in the safest way, applying the changes to an environment then running a comprehensive suite of automated tests before progressing and applying to the next environment. In this way, the production environment of sites receives only assured upgrades to its foundational code.

The high-level overview of the project is split into two parts:

  1. The first part of the project is to improve testing coverage across our sites. This is done by creating clear strategy and standards around testing requirements for websites, and then working with the SCOs and Operations teams to educate development teams and enforce those requirements.

    Part of this step has also been to build tools that make it easier for teams to write, maintain and run tests.

    This first part is critical and will define success of the whole program.
  2. The second part is focuses on building the automation system that will, step-by-step, upgrade a site on each environment.

The project is progressing well. The first part of the project was actively worked on last year. The strategy and standards are set, the tools released. The focus is currently on educating the teams.

The second part is also well underway. The initial implementation is in place and 10 production external Edison Custom sites are in the pilot program, which means they have been automatically upgraded with every new release of Edison Base for a few months now.

Benefits this work brings to Pfizer

It will:

  • Provide savings to teams who will benefit from having increased automatic testing coverage
  • Assure our Edison Custom sites are more secure than ever before
  • Allow our Operations teams to focus on more productive, business-essential tasks

Looking ahead at 2021

At the moment, Edison’s focus is on growing the pilot program by adding more and more Edison Custom sites. In order to join the program, the site must be on the latest version of Edison Base and have a good amount of test coverage. We are currently working with the Digital Fortress and Edison Custom Migration teams who are moving sites from Drupal 7 and 8 to Drupal 9. These sites are on the latest Edison Base version and have strong test coverage, so we can add them directly into Automated Upgrades.

Edison is committed to continuous improvement, and constantly monitors how the system is performing and if any tweaks are needed to improve it.
